script
첫 번째 head태그에 script가 있으면 html을 parsing 하다가 script를 만나면 main.js를 다운받아 실행하고 다시 html을 parsing함.
단점 -js파일이 크면 많은 시간이 걸림
async
html을 parsing 하다가 script를 만나면 main.js다운만 받고, 계속 html을 parsing 하다가 다운로드가 되면 parsing을 멈추고 script를 실행하고 나머지 html을 파싱함.
장단점 -body 하단에 넣는 것보다 다운로드 받는 시간은 절약할 수 있으나 다 다운 받기 전에 script가 실행될 수 있으므로 DOM요소를 조작할려고 할 때 html의 파싱이 되기 전에 스크립트가 실행이 되면 원하는 것을 실행할 수 없을 수 있음
defer
html을 parsing 하다가 script를 만나면 main.js다운만 받고 계속 html을 parsing 파싱을 다 하고 script를 실행함. 가장 좋은 방법임

바닐라 자바스크립트로 개발할 땐 'use strict'; - 상식적인 범위 안에서 자바스크립트를 이용할 수 있게 해줌 성능개선까지... 유연하게 만들어져 있기 때문에 비상식적인 것들이 생기지 않음
